Visual Studio Code設定メモ¶

インストール¶

Visual Studio Codeのサイトに行ってダウンロード。 ファイルコンテクストメニューのチェックとエディタの登録にチェックを入れる。 あとは多分デフォルト設定でよい。

拡張機能¶

拡張機能は左の四角いパネルのアイコンを押して(Ctrl+Shft+X)管理ができる。 アイコンをもう一度押すとパネルが再度非表示になる。

日本語化¶

以下の拡張機能を検索、インストールする。 Japanese Language Pack for Visual Studio Code

シンタックスハイライト¶

開いているファイルの拡張子に応じて自動的に拡張機能を推奨してくるので入れる。

各種設定¶

「ファイル-基本設定-設定」または「左下の歯車-設定」を押して設定していく。 設定画面の右上の「{}」を押して開くsettings.jsonファイル（多分右側）をいじってもよい。

インデントの設定をファイルから自動で読み込むか タブキーを押した時に半角スペースに変換せずタブのまま入力する スペースを表示するか(boundaryは単語間の単独半角スペースは非表示) タブに相当する半角スペースの個数 改行コードなど制御文字の表示（CRLFかLFかの確認）

"editor.tabSize": 4, "editor.detectIndentation": false, "editor.insertSpaces": false, "editor.renderWhitespace": "boundary", "editor.renderControlCharacters": true,

タブの設定はデフォルトの半角スペースのほうがよいかもしれない。 その他いろいろ設定があるので適宜確認する。

スペースの着色¶

拡張機能「Whitespace+」を入れる。 「表示-コマンドパレット」(Ctrl＋Shift+P)で「Whitespace+ Config」と入力して実行 以下設定の抜粋。スペースは単語間の単独半角スペースは着色なしにしている。

"mode": "all", "autoStart": true,

"name": "space", "enabled": true, "pattern": "\s{2,}",

"name": "tab", "enabled": true,

全角スペース表示¶

拡張機能「zenkaku」を入れる。

改行コード表示¶

拡張機能「line-endings」を入れる。

マークダウンPDF出力¶

マークダウンの文書をPDFとして出力する。 pandocを使う方法があるらしいが、未着手。 拡張機能「Markdown PDF」を入れる。右クリックしてMarkdown PDF:Export(pdf)を選択するとマークダウンファイルと同じ場所に出力される。

PDFの設定例。 設定画面の右上の「{}」を押して開くsettings.jsonファイルに以下の記述を加える。:

"markdown-pdf.styles": ["markdown-pdf.css",],
"markdown-pdf.headerTemplate": "<span class='title' style='display: none;'></span>"

以下の場所にmarkdown.cssをコピーしてmarkdown-pdf.cssという名前にして置く。 C:Users[username].vscodeextensionsyzane.markdown-pdf-1.2.0styles

markdown-pdf.cssのフォントの設定を変更する。(中国語フォントを避けるため日本語フォントの指定は必要):

body {
    font-family:  "Meiryo", "Segoe WPC", "Segoe UI", "SFUIText-Light", "HelveticaNeue-Light", sans-serif, "Droid Sans Fallback";
}
code {
    font-family:  "Meiryo", "Segoe WPC", "Segoe UI", "SFUIText-Light", "HelveticaNeue-Light", sans-serif, "Droid Sans Fallback";
}

PDFにて改ページを行いたい場合は以下の記述をマークダウンの該当箇所に追加する。 <div style="page-break-before:always"></div>
